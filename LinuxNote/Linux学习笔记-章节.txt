2-1 虚拟机的安装
1. VMWare简介
VMWare是一个虚拟PC的软件，可以在现有的操作系统上虚拟出一个新的硬件环境，相当于模拟出一台新的PC，
依次来实现在一台机器上真正同时运行两个独立的操作系统。
使用VMWare8，占用空间较小。

VMWare主要特点：
    -不需要分区或重新开机就能在同一台PC上使用两种以上的操作系统
    -本机系统可以与虚拟机系统网络通信
    -可以设定并且随时修改虚拟机操作系统的硬件环境
    虚拟机有快照功能，如果系统崩溃了，系统可以恢复到快照时的状态。

2、建议的VMWare配置
    CPU：建议主频为1GHz以上
    内存：建议1GB以上
    硬盘：建议分区空闲空间8G以上

2-3 系统分区之分区与格式化
1. 磁盘分区
    磁盘分区是使用分区编辑器（partition editor）在磁盘上划分几个逻辑部分。（不是物理上的划分，只是概念上的分区）
    碟片一旦划分成数个分区（Partition），不同类的目录与文件可以存储进不同的分区。

    分区类型
        主分区：最多只能有4个。
        扩展分区：
            做多只能有1个。
            主分区加扩展分区最多有4个。
            扩展分区不能写入数据，只能包含逻辑分区。
        逻辑分区

2. 格式化
    格式化（高级格式化）又称逻辑格式化，它是指根据用户选定的文件系统（如FAT16、FAT32、NTFS、EXT2、EXT3、EXT4等），
    在磁盘的特定区域写入特定数据，在分区中划出一片用于存放文件分配表、目录表等用于文件管理的磁盘空间。

    NTFS支持更大的空间和更大的单个文件大小。
    格式化就是写入文件系统，它包括两部分，一是把磁盘分为等大小的数据块；二是在分区中建立二维表格，记录每个文件的ID号、修改时间和权限。
    默认数据块（block）的大小是4kB。
    格式化会将数据清空，这只是它的效果，它的目的是写入文件系统。

2-4 分区之分区设备文件名与挂载
3. 硬件设备文件名
    硬件                      设备文件名
    IDE硬盘                   /dev/hd[a-d]
    SCSI/SATA/USB硬盘         /dev/sd[a-p]
    光驱                      /dev/cdrom或/dev/hdc
    软盘                      /dev/fd[0-1]
    打印机（25针）             /dev/lp[0-2]
    打印机（USB）              /dev/usb/lp[0-15]
    鼠标                      /dev/mouse

    /表示根目录
    dev目录下存放的硬件设备文件

    分区设备文件名
        设备文件名
            /dev/hda1（IDE硬盘接口）
            /dev/sda1（SCSI硬盘接口、SATA硬盘接口）

    分区 --> 格式化 --> 分区设备名 --> 挂载点（相当于Windows的盘符）
    1、2、3、4四个分区号只能给主分区，逻辑分区是从5开始的。

4、挂载
    必须挂载
        /（根分区）
        swap分区（交换分区，内存2倍，不超过2GB）
    推荐分区
        /boot（启动分区，200MB）

总结：
分区：把大硬盘分为小的逻辑分区
格式化：写入文件系统
分区设备文件名：给每个分区定义设备文件名
挂载：给每个分区分配挂载点

命令格式
    命令 [选项] [参数]
注意： 个别命令使用不遵循此格式
      当有多个选项时，可以写在一起，简化选项与完整选项，选项可以调整命令的功能
        -a 等于 --all，并不是所有命令都支持 -a 选项

查询目录中内容：ls
ls [选项] [文件或目录]
选项：
    -a 显示所有文件，包括隐藏文件（all）
    -l 显示详细信息 （long）
    -d 查看目录属性
    -h 人性化显示文件大小
    -i 显示inode（文件的id号）

[root@localhost bin]# ls -l
dr-xr-xr-x.   2 root root  4096 10月  5 04:55 bin
d 文件类型（- 文件， d 目录， l 软链接文件）
    Linux一共有7种文件，还包括块设备文件、字符设备文件、套接字文件和管道文件，通过权限的第一个字符区分
r-x，u所有者的权限
r-x，g所属组的权限（相同身份或者相同权限的人分为一组）
r-x，o其他人的权限
r-read-4
w-write-2
x-execute-1

dr-xr-xr-x（权限）.   2（引用计数） root（所有者） root（所属组）  4096（文件大小） 10月  5 04:55（最后一个修改时间） bin（文件名）
4096-文件大小， ls -lh（h代表human，为了人们更好的理解数据的含义，只要包含文件大小的命令均可以使用-h命令）

[选项]，是为调整命令的功能的。
[root@localhost bin]# ls -lh cat
-rwxr-xr-x. 1 root root 44K 11月 22 2013 cat
[root@localhost bin]# ls（命令） -lh（选项） cat（参数）

Linux中以.开头的文件是隐藏文件，一般是系统文件，可以提示用户不要轻易修改它。

[root@localhost bin]# ls -i

ll 是 ls -l 命令的别名

3-4 Linux链接命令
链接命令：ln

ln -s [原文件] [目标文件]
    link
    功能描述：生成链接文件
    选项：-s   创建软链接（soft）

硬链接与软链接：
硬链接特征：（一个真实文件的多个别名）
1 拥有相同的i节点和存储block块，可以看做是同一个文件
2 可通过i节点识别
3 不能跨分区（可以把硬链接看做一个教室的两个门，这个教室不能同时在两栋楼中）
4 不能针对目录使用

软链接特征：（推荐使用软链接）
1 类似windows快捷方式
2 软链接拥有自己的I节点和Block块，但是数据块中只保存原文件的文件名和I节点号，并没有实际的文件数据
3 lrwxrwxrwx l 软链接
    软链接文件权限都为 rwxrwxrwx
4 修改任意文件，另一个都改变
5 删除原文件，软链接不能使用

创建软链接时，原文件一定要写绝对路径，除非和目标文件在同一个目录下

4-1 文件搜索命令locate
locate命令格式
locate 文件名
    在后台数据库中按文件名搜索，搜索速度更快
    /var/lib/mlocate，是locate命令所搜索的后台数据库，不用遍历整个文件系统

[root@localhost home]# locate liyazhou
locate: can not stat () `/var/lib/mlocate/mlocate.db': 没有那个文件或目录
[root@localhost home]# updatedb
[root@localhost home]# locate liyazhou

优点，搜索速度快
缺点，只能按照文件名称搜索，功能单一，与find互补
一般情况下，按文件名搜索情况较多，所以locate较常用

4-3 find命令
文件搜索命令find
功能强大，参数较多，学习起来复杂，此处仅仅学习常用的操作

find [搜索范围] [搜索条件]
    find / -name install.log
    避免大范围搜索，会非常耗费系统资源
    find是在系统中搜索符合条件的文件名。如果需要匹配，使用通配符匹配，通配符是完全匹配。

[root@localhost ~]# find / -name liyazhoutemp
/home/liyazhoutemp

[root@localhost ~]# find / -name liyazhou*
/home/liyazhoutemp

Linux中的通配符
    *   匹配任意内容
    ?   匹配任意一个字符
    []  匹配中括号内任意的一个字符

不区分大小写
find /root -iname install.log

按照所有者搜索
find /root -user root

查找没有所有者的文件
find /root -nouser
没有所有的文件，一般是垃圾文件
一是，内核产生的文件可能没有所有者，比如在/sys
二是，外来的文件，可能没有所有者
除了两者之外，没有所有者的文件可以看做是垃圾文件

查找10天前修改的文件（处理日志文件常用）
find /var/log/ -mtime +10
    -10     10内修改的文件
     10     10天当前修改的文件
    +10     10天前修改的文件

    atime   文件访问时间
    ctime   改变文件属性
    mtime   修改文件内容

查找文件大小是25kB的文件
find . -size 25k
    -25k    小于25kB的文件
    25k     等于25kB的文件
    +25     大于25kB的文件
单位
KB——k（小写）
MB——M（大写）

查找inode是262422的文件
find . -inum 262422
    跟 ls -i 配合使用
    ls -i 查看inode
    find dir -inum inodeid 已知文件编号查文件

查找/etc/目录下，大于20KB并且小于50KB的文件
find /etc -size +20k -a -size -50k
    -a and 逻辑与，两个调价都满足
    -o or  逻辑或，两个条件满足一个即可

查找/etc/目录下，大于20KB并且小于50KB的文件，并显示详细信息
find /etc -size +20k -a -size -50k -exec ls -lh {} \;
    对搜索结果执行操作 -exec/-ok 命令 {} \;

9-1 Shell概述

1. Shell是什么
Shell 是一个命令行解释器，它为用户提供了一个向Linux内核发送请求以便运行程序的界面系统级程序，
用户可以用Shell来启动、挂起、停止甚至编写一些程序。

Shell还是一个功能相当强大的编程语言，易编写，易调试，灵活性较强。
Shell是解释执行的脚本语言，在Shell中可以直接调用Linux系统命令。

2.Shell的分类
Bourne Shell：从1979起Unix就开始使用Bourne Shell, Bourne Shell的主文件名为sh。（不支持历史命令）
C Shell：C Shell主要在BSD版的Unix系统中使用，其语法和C语言相类似而得名。

Shell的两种主要语法类型有Bourne和C，这两种语法彼此不兼容。
Bourne家族主要包括sh、ksh、Bash、psh、zsh；
C家族主要包括：csh、tcsh。

Bash：Bash与sh兼容，现在使用的Linux就是使用Bash作为用户的基本Shell。
Linux中的标准Shell是Bash。

3. Linux支持的Shell
echo $SHELL
/etc/shells

9-5 输出重定向

1. 标准输入输出

设备      设备文件名       文件描述符       类型
键盘      /dev/stdin      0               标准输入
显示器    /dev/stdout     1               标准输出
显示器    /dev/stderr     2               标准错误输出

2. 输出重定向

标准输出重定向：
命令 > 文件，以覆盖的方式，把命令的正确输出输出到指定的文件或设备当中
命令 >> 文件，以追加的方式，把命令的正确输出输出到指定的文件或设备当中

标准错误输出重定向
错误命令 2>文件（注意，此处没有空格），以覆盖的方式，把命令的错误输出输出到指定的文件或设备当中
错误命令 2>>文件（注意，此处没有空格），以追加的方式，把命令的错误输出输出到指定的文件或设备当中

[root@localhost ~]# ll -h > /root/tmp/redirectfile.zhou
[root@localhost ~]# cat /root/tmp/redirectfile.zhou

正确输出和错误输出同时保存（最为常用）
命令 > 文件2>&1，以覆盖的方式，把正确输出和错误输出都保存到同一个文件当中
命令 >> 文件 2>&1，以追加的方式，（掌握）
命令 &>文件，以覆盖的方式，
命令 &>>文件，以追加的方式，（掌握）
命令>>文件1 2>>文件2，把正确的输出追加到文件1中，把错误的输出追加到文件2中（掌握）

3. 输入重定向（相对于输出重定向，输出重定向较少使用）
wc [选项] [文件名]
    选项：
        -c：统计字节数
        -w：统计单词书
        -l：统计行数

命令 < 文件把文件作为命令的输入
命令 << 标识符
标识符把标识符之间内容作为命令的输入

9-6 管道符（非常常用）

1. 多命令顺序执行
符号 ;
格式 命令1; 命令2
作用 多个命令顺序执行，命令之间没有任何逻辑关系

符号 &&
格式 命令1&&命令2
作用 逻辑与，短路与操作

符号 ||
格式 命令1||命令2
作用 逻辑或，短路或

2. 管道符
命令格式：命令1 | 命令2
作用：命令1的正确输出作为命令2的操作对象
注意，命令2一定能够操作命令1的输出
[root@localhost ~]# ls -lh | more
more 分屏查看文件

[root@localhost ~]# netstat -an | grep ESTABLISHED
tcp        0      0 192.168.13.215:22           192.168.13.2:54033          ESTABLISHED

[root@localhost ~]# netstat -an | grep ESTABLISHED | wc -l
1

wc -l 文件，统计行号
